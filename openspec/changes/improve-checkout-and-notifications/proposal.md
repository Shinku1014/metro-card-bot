## Why

当前系统在出站时需要手动选择优惠券，即使只有一种可用，这增加了不必要的操作。此外，UI 显示不够直观（例如使用 `-2` 代替中文），且缺乏对误操作的撤销支持及长时间未出站的提醒，可能导致状态不同步。

## What Changes

- **自动选择优惠券**: 修改出站逻辑，若当前仅有一种优惠券（A 或 B）可用（剩余数量 > 0 且今日未用），则自动消耗并提示，无需弹出选择菜单。
- **UI 文本优化**: 将卡片列表中显示的 `(五折:5 -2:3)` 优化为 `(五折: 5 减二: 3)`，其中 -2 改成中文 减二，冒号后加空格，提升易读性。
- **撤销进站**: 在卡片详情或操作中增加“刚才点错了/撤销进站”功能，将状态恢复为 `idle` 而不消耗优惠券。
- **超时出站提醒**: 增加后台检测机制，若用户进站超过 210 分钟（默认值，可配置）未出站，主动推送通知，并提供“刚才点错了”和“现在出站并选择优惠券”两个选项。

## Capabilities

### New Capabilities
- `auto-coupon-selector`: 实现在出站时根据可用优惠券数量自动决策的逻辑。
- `ui-text-formatting`: 统一优化全系统的优惠券显示文案。
- `undo-check-in`: 提供无需消耗优惠券即可重置卡片状态的功能。
- `checkout-reminder`: 实现基于时间的超时检测和交互式推送通知。

## Impact

- **src/index.ts**: 需要修改卡片点击的处理逻辑 (`card_${cardId}`)，更新 UI 渲染函数，并增加新的 Action 处理（如 `undo_checkin`）。
- **src/dataManager.ts**: 可能需要增加获取所有进站状态卡片的方法，以供提醒机制使用。
- **src/types.ts**: 可能需要为提醒机制增加辅助字段或配置常量。
- **项目结构**: 引入定时任务（如 `setInterval` 或 `cron`）来驱动后台提醒。
